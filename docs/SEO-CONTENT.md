### Canonical URLs

**Implementation Rules:**

1. Always use HTTPS
2. Always use www or non-www consistently (prefer non-www)
3. Remove trailing slashes
4. Remove query parameters for canonical

```html
<!-- Homepage -->
<link rel="canonical" href="https://citegenerator.org" />

<!-- Style pages -->
<link rel="canonical" href="https://citegenerator.org/apa-citation-generator" />

<!-- Tutorial pages -->
<link rel="canonical" href="https://citegenerator.org/how-to-cite-website" />

<!-- Self-referencing on all pages -->
<link rel="canonical" href="[current-page-url]" />
```

### Core Web Vitals Targets

| Metric                              | Target  | Strategy                                                                       |
| ----------------------------------- | ------- | ------------------------------------------------------------------------------ |
| **LCP** (Largest Contentful Paint)  | < 2.5s  | Preload hero fonts, lazy load below-fold images, CDN for static assets         |
| **FID** (First Input Delay)         | < 100ms | Defer non-critical JS, code-split citation logic, use Web Workers for parsing  |
| **CLS** (Cumulative Layout Shift)   | < 0.1   | Reserve space for ads, define image dimensions, avoid layout-shifting elements |
| **INP** (Interaction to Next Paint) | < 200ms | Optimize event handlers, debounce input, minimize main thread work             |
| **TTFB** (Time to First Byte)       | < 600ms | Edge caching, server-side rendering, efficient database queries                |

**Performance Checklist:**

- [ ] Enable Brotli compression
- [ ] Implement HTTP/2 or HTTP/3
- [ ] Use `<link rel="preconnect">` for external domains
- [ ] Preload critical fonts: `<link rel="preload" as="font">`
- [ ] Lazy load images with `loading="lazy"`
- [ ] Defer non-critical JavaScript
- [ ] Inline critical CSS
- [ ] Use responsive images with `srcset`
- [ ] Implement service worker for offline support
- [ ] Cache API responses for common lookups (DOI, ISBN)

# How to Cite a Website in APA Format

**Last Updated:** January 2025
**Style:** APA 7th Edition

Learn how to properly cite a website in APA format with step-by-step instructions, examples, and a free citation generator.

## Quick Reference

**Reference List Format:**
Author, A. A. (Year, Month Day). _Title of page_. Site Name. URL

**In-Text Citation:**
(Author, Year) or Author (Year)

[Generate APA Website Citation Button]

---

## Step-by-Step Guide

### Step 1: Identify the Author

Look for the author's name on the webpage. Check:

- The byline under the title
- The "About" section
- The page footer

**If no author is found:**

- Use the organization name (e.g., World Health Organization)
- If no organization, start with the page title

### Step 2: Find the Publication Date

Look for when the page was published or last updated. Common locations:

- Below the title
- At the bottom of the article
- In the URL (sometimes)

**If no date is found:**

- Use "n.d." (no date)

### Step 3: Note the Page Title

Copy the exact title of the page you're citing.

- Use italics for the title
- Capitalize only the first word and proper nouns

### Step 4: Identify the Website Name

This is the overall website, not the page title.

- Write in plain text (not italics)
- Omit if same as author

### Step 5: Copy the URL

- Use the full URL
- Remove tracking parameters (?utm_source=...)
- Don't include a period after the URL

## Frequently Asked Questions

**Do I need the access date for websites?**
No. APA 7th edition removed the retrieval date requirement for most online sources. Only include it for pages that change frequently (like social media or wikis).

**How do I cite a website with no author?**
Start the citation with the title of the page in italics, followed by the date and other elements.

**Should I use http:// or https://?**
Use whatever the URL actually shows. Most sites now use https://.
